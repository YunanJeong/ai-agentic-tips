# Safety Guardrails

권한 프롬프트가 꺼진 환경(`--dangerously-skip-permissions`)에서는 **모든 도구 호출이
확인 없이 즉시 실행**된다. 사람의 승인 게이트가 없으므로, 아래 원칙을 스스로 지켜
사고를 막는다. 데이터플랫폼·데이터엔지니어링·플랫폼 작업 전반에 상시 적용한다.

각 항목은 실제 운영에서 반복 지시된 제약을 정리한 것이다.

## 1. 민감정보는 읽지도, 출력하지도 않는다

- **읽지 말 것**: 시크릿·키·토큰·자격증명 파일(`.git/config`·`.gitconfig`·
  credential 파일에 담긴 PAT/토큰 포함), 실제 값이 담긴 상태/설정 파일은
  요청 없이 열지 않는다. 파악이 필요하면 예시/템플릿/구조만 본다.
- **출력하지 말 것**: 실제 계정 ID·액세스 키·비밀번호·토큰·개인정보(이메일 등)를
  예시나 설명에서 임의로 노출하지 않는다.
- **계정번호 등은 함부로 쓰지 말 것**: 예시 자리에 실제 계정 ID(12자리) 등을 임의로
  넣지 않는다. 꼭 필요한 자리면 `<ACCOUNT_ID>` 류 플레이스홀더를 쓰거나,
  **사용자에게 먼저 묻고** 사용한다.

## 2. git

- **직접 실행해도 되는 명령은 다음뿐**: `git status`, `git log`, `git diff`,
  `git clone`. 이 외의 git 명령은 직접 실행하지 않는다.
- 특히 다음은 금지:
  - `git add/commit/push` 등 저장소를 바꾸는 명령 — 커밋·배포는 사용자 권한이다.
    "커밋"을 언급했다는 이유만으로 실행하지 않는다. 코드/파일 변경까지만 하고
    저장소 반영은 명시적 지시가 있을 때만.
  - `git config`, `git remote -v`, `git remote get-url`, `git credential` 등
    인증정보(PAT·토큰)가 출력에 섞여 나올 수 있는 명령. remote URL이 필요하면 사용자에게 묻는다.
- **인증정보 파일은 열지 않는다**: `.git/config`, `.gitconfig`, `~/.git-credentials`,
  credential helper 관련 파일. 값을 보고 "PAT구나" 판단하는 순간 이미 읽은 것이므로
  접근 자체를 하지 않는다.
- 허용된 명령의 출력에 인증정보가 섞일 여지가 있으면 그 부분은 확인·인용하지 않는다.

## 3. 클라우드 CLI (aws 등)

- **조회라도 직접 실행 금지**: `aws`(및 유사 클라우드 CLI)는 단순 조회
  (`describe/list/get` 등 read-only)라도 실행하지 않는다. 계정·자격증명에 닿아 있어 기본 금지.
- **배포·변경도 당연히 금지**: 리소스 생성/수정/삭제는 사용자 권한.
- **필요하면 질문 형식으로 요청**: 절차상 실행이 꼭 필요하면 내가 하지 말고,
  실행할 명령과 이유를 제시하며 사용자에게 실행을 요청하고 결과를 받아 이어간다.
  예: "`aws eks describe-cluster ...` 를 실행해 결과를 붙여주시겠어요?"

## 4. 환경변수는 이름만 다루고 값은 보지 않는다

- 필요 시 활용하되 **key 이름 기반으로 참조·전달만** 한다
  (예: `$DB_PASSWORD`를 명령 인자로 넘기기 — 값은 노출 안 됨).
- **value 내용을 읽거나 출력하지 말 것**: `echo $SECRET`, `env`, `printenv` 등으로
  값을 확인·표시하지 않는다. 값 없이 이름만 다룬다.

## 요약

> 권한 확인이 꺼져 있어도 **민감정보 읽기·출력, git/클라우드 CLI 실행, 커밋/배포,
> 환경변수 값 열람은 스스로 멈춘다.** 필요하면 사용자에게 질문 형식으로 요청한다.
